Why XGBoost + DNN?
XGBoost — highest recall (81.8%). Only 51 missed churners out of 280. Best at catching at-risk customers early.
DNN — captures complex nonlinear patterns tree models miss. Adds signal on borderline customers.
Ensemble — averaging probabilities reduces variance. Disagreement >10pp → flag for human review.

Business insights from the model
#1 Contract type — month-to-month customers churn 3× more than 2-year contract holders. Top priority for upsell.
#2 Tenure — strongest protective factor. After 24 months, churn risk drops sharply. Focus spend on 0–12 month customers.
#3 Monthly charges — higher bills = higher churn. Bundle extra services to justify the cost.
#4 Fiber optic — highest churn service type. Customers feel they pay too much for too little.
#5 Online security + tech support — customers with both services churn far less. Clear upsell opportunity.
#6 Electronic check — e-check customers churn 2× vs auto-pay. Incentivise switching to bank transfer.
Deploy guide
From saved models → FastAPI → this dashboard · All files downloadable above
1
Models saved ✓
All 5 models already saved from notebook: lgb_model.txt, xgb_model.json, dnn_model.keras, scaler.pkl, metadata.json
saved_models/ ✅ (687KB DNN · 316KB XGB · 92KB LGB)
2
Create virtual environment
Creates an isolated Python environment for this project.
python -m venv venv # Windows: venv\Scripts\activate # Mac/Linux: source venv/bin/activate
3
Install dependencies
Installs FastAPI, LightGBM, TensorFlow, XGBoost and all required libraries.
pip install -r requirements.txt # Takes 3–10 minutes first time
4
Run FastAPI server
Starts the prediction API on port 8000. Keep this terminal open.
uvicorn fastapi_backend:app --reload --port 8000 # API docs → http://localhost:8000/docs # Health → http://localhost:8000/health
5
Open dashboard.html
Double-click dashboard.html — works immediately with simulated data. To use real models, set API_URL in the file.
// In dashboard.html, line ~10 const API_URL = "http://localhost:8000";
